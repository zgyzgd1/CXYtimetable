## 🔴 阻塞级（构建必然失败）

### 1. Kotlin Compose Plugin `2.3.10` 不存在
- **文件**: `build.gradle.kts` (root)
- **置信度**: 90
- **问题**: Kotlin 2.3.x 未发布（最新稳定版 2.1.x），Compose compiler plugin 版本必须匹配 Kotlin 版本，构建将失败。
- **修复**: 使用已知匹配版本（如 Kotlin `2.1.20` + compose-plugin `2.1.20`）。

### 2. KSP `2.3.6` 不存在
- **文件**: `build.gradle.kts` (root)
- **置信度**: 90
- **问题**: KSP 版本跟踪 Kotlin 发布，2.3.x 不存在，`com.google.devtools.ksp:2.3.6` 无法解析。
- **修复**: 使用 `2.1.20-1.0.29` 或匹配 Kotlin 版本的对应 KSP 版本。

### 3. buildToolsVersion = `37.0.0` 不存在
- **文件**: `app/build.gradle.kts`
- **置信度**: 85
- **问题**: Build-tools 37.0.0 对应不存在的 SDK。compileSdk = 36 应匹配 buildTools `36.0.0`（或省略行让 AGP 自动选择）。

### 4. 多个依赖版本不存在
- **文件**: `app/build.gradle.kts`
- **置信度**: 85
- **问题**: 以下版本在 Maven 仓库中不存在，构建将失败：
  - `activity-compose:1.13.0`（最新已知: 1.9.3）
  - `core-ktx:1.18.0`（最新已知: 1.15.0）
  - `lifecycle-runtime-compose:2.10.0`（最新已知: 2.8.7）
  - `lifecycle-viewmodel-compose:2.10.0`（同上）
  - `work-runtime-ktx:2.10.1`（最新已知: 2.9.1）
  - `room:2.8.4`（最新已知: 2.6.1）

### 5. gradle.properties 含 U+FEFF BOM 字符
- **文件**: `gradle.properties`
- **置信度**: 85
- **问题**: BOM 字符导致 key 变成 `\ufeffandroid.useAndroidX` 而非 `android.useAndroidX`，**AndroidX 被静默禁用**，编译 AndroidX API 时产生编译错误。
- **修复**: 保存为 UTF-8 无 BOM。

## 🟡 功能性 Bug

### 10. 全天事件 (VALUE=DATE) 解析为午夜 1 小时
- **文件**: `IcsImport.kt` / `IcsShared.kt`
- **置信度**: 90
- **问题**: `DTSTART;VALUE=DATE:20260907` 解析为当天 00:00，`DTEND` 缺失时回退 `start.plusHours(1)` 创建 1 小时事件。传入 Google Calendar 等将显示错误时间。

### 11. WebView 离开组合时未调用 destroy()
- **文件**: `JwWebView.kt`
- **置信度**: 90
- **问题**: `AndroidView` 创建 WebView 实例，组合离开时 `AndroidView` 仅从父容器移除视图，未调用 `WebView.destroy()`，泄漏原生渲染引擎和 GPU 内存。

### 12. 复杂循环规则被忽略
- **文件**: `IcsImport.kt`
- **置信度**: 85
- **问题**: 非每周循环分支仅按 freq 间隔递增。`FREQ=MONTHLY;BYDAY=2TU`（每月第二个周二）退化为简单周期事件，产生错误日期。

### 13. EXIF 方向元数据未处理
- **文件**: `BackgroundImageManager.kt`
- **置信度**: 85
- **问题**: `BitmapFactory.decodeStream` 不应用 EXIF 旋转。相机照片含方向 EXIF 标签时，背景图可能旋转 90°/180°/270°。

### 14. RRULE 无 COUNT/UNTIL
- **文件**: `IcsExport.kt`
- **置信度**: 85
- **问题**: 导出循环事件仅含 `FREQ=WEEKLY / INTERVAL=2 / BYDAY=xx` 无终止条件。导入 Google Calendar/Outlook/Apple Calendar 后无限重复。

### 15. weakKeys Map 覆盖导致去重丢失
- **文件**: `HebauCourseParser.kt`
- **置信度**: 80
- **问题**: 多课程共享相同 `weakDedupeKey` 但不同 `dedupeKey` 时，后写入覆盖前者的映射，产生重复课程。

### 16. distinctBy { it.id } 静默丢弃有效事件
- **文件**: `IcsImport.kt`
- **置信度**: 85
- **问题**: 当 iCalendar 含多个共享相同 UID 的 VEVENT（如通过 RECURRENCE-ID 覆盖特定实例），重复条目被静默丢弃。

### 17. computeTriggerAtMillis 静默吞异常
- **文件**: `CourseReminderScheduler.kt`
- **置信度**: 80
- **问题**: `runCatching` 捕获所有异常返回 null，数据损坏导致计算失败时无法区分"无未来事件"和"计算异常"。

### 18. 默认 requestCode=0 导致通知互相覆盖
- **文件**: `CourseReminderReceiver.kt`
- **置信度**: 85
- **问题**: `getIntExtra(EXTRA_REQUEST_CODE, 0)` 中 Intent 缺失该 Extra 时，所有提醒使用 `requestCode=0`，`NotificationManager.notify(0, ...)` 只显示最后一条通知。
